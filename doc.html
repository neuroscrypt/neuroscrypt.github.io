Форматы данных для нейрокоммуникации

Документация по структуре файлов и протоколам обмена эмоциональными состояниями и мысленными командами с использованием нейрогарнитуры OpenBCI

Введение

Эта документация описывает форматы данных и протоколы для обмена эмоциональными состояниями между пользователями нейрогарнитур. Система позволяет кодировать, передавать и визуализировать эмоциональные состояния, полученные на основе анализа ЭЭГ-сигналов.

Примечание: В текущей версии система работает с базовыми эмоциональными состояниями. В перспективе планируется расширение для передачи более сложных когнитивных паттернов.

Формат файла эмоции

Файлы эмоций содержат метаданные и параметры эмоционального состояния для передачи между устройствами.

Структура JSON:

{
  "version": "1.0",
  "timestamp": "2023-11-15T14:30:00Z",
  "user_id": "user_12345",
  "session_id": "session_abc123",
  "emotion": {
    "primary": "joy",
    "intensity": 0.85,
    "valence": 0.9,
    "arousal": 0.7,
    "confidence": 0.78
  },
  "eeg_data": {
    "channels": 8,
    "sample_rate": 250,
    "features": {
      "alpha_power": [0.45, 0.38, ...],
      "beta_power": [0.32, 0.29, ...],
      "theta_power": [0.21, 0.18, ...]
    }
  },
  "visualization": {
    "color": "#FFD700",
    "pattern": "pulsing",
    "intensity": 0.8
  }
}

Формат файла сессии

Файлы сессии содержат данные о всей сессии записи, включая временные метки и последовательности эмоциональных состояний.

Структура JSON:

{
  "version": "1.0",
  "session_id": "session_abc123",
  "user_id": "user_12345",
  "start_time": "2023-11-15T14:00:00Z",
  "end_time": "2023-11-15T14:30:00Z",
  "device_info": {
    "type": "OpenBCI Ultracortex",
    "channels": 16,
    "sampling_rate": 250
  },
  "emotional_timeline": [
    {
      "timestamp": "2023-11-15T14:05:23Z",
      "emotion": "calm",
      "intensity": 0.6
    },
    {
      "timestamp": "2023-11-15T14:07:45Z",
      "emotion": "joy",
      "intensity": 0.8
    }
  ],
  "metadata": {
    "environment": "quiet_room",
    "activity": "meditation"
  }
}

Протокол обмена в реальном времени

Для обмена эмоциональными состояниями в реальном времени используется WebSocket-протокол со следующей структурой сообщений:

Структура сообщения:

{
  "type": "emotion_update",
  "sender": "user_12345",
  "timestamp": 1699986600000,
  "data": {
    "emotion": "joy",
    "intensity": 0.85,
    "color": "#FFD700",
    "animation": "pulse"
  }
}

Типы сообщений:

  • emotion_update - обновление эмоционального состояния
  • session_start - начало сессии
  • session_end - завершение сессии
  • device_status - статус подключения устройства
  • error - сообщение об ошибке

Визуализация эмоций

Каждое эмоциональное состояние связано с определенной визуализацией для отображения на устройствах:

Цветовая схема

  • Радость: Желтый (#FFD700)
  • Грусть: Синий (#3498DB)
  • Спокойствие: Зеленый (#2ECC71)
  • Страх: Фиолетовый (#9B59B6)
  • Гнев: Красный (#E74C3C)

Анимационные паттерны

  • Пульсация - для спокойных состояний
  • Бегущая волна - для радости и возбуждения
  • Мерцание - для удивления
  • Плавное изменение - для переходов между состояниями
  • Быстрое мигание - для интенсивных эмоций

Перспективы развития

В будущем система может быть расширена для передачи более сложных когнитивных состояний:

  • Мысленные команды - передача намерений и простых команд
  • Когнитивная нагрузка - оценка уровня умственного напряжения
  • Визуальные образы - передача простых геометрических форм
  • Сенсорные ощущения - ассоциации с тактильными ощущениями
  • Сложные эмоциональные паттерны - смешанные и тонкие эмоции

Важно: Для реализации передачи мыслей потребуются дополнительные исследования и более сложные алгоритмы машинного обучения для декодирования нейронных паттернов.
